guardd-suite v0.1.0 使用说明

一、统一安装命令
curl -fsSL https://github.com/1227cwx/tsycdn-guardd-release/releases/latest/download/install.sh | bash

脚本启动后选择：
1) 安装 guardd 节点
2) 安装 guardd-center 中心端
3) 安装 guardd-test 节点测试工具
0) 退出

二、guardd-center 安装说明
- 安装时需要填写 Web 监听 IP/端口、HTTP/HTTPS、自定义管理员账号密码。
- 必须配置 MySQL 和 Redis；安装脚本会先测试 MySQL 建表/读写，再测试 Redis PING/写入/读取/删除，全部通过才会继续。
- 安装时会询问是否安装 GeoIP 离线资源，默认安装；资源包括 DB11 城市库、ASN 库、中文地区、国旗、世界地图。
- GeoIP 资源默认安装到：/var/lib/guardd-center/geoip
- systemd 会设置 GUARDD_CENTER_GEOIP_DIR=/var/lib/guardd-center/geoip。

三、guardd 节点安装说明
- 安装时设置节点名称、分组、API 监听 IP/端口、Key、允许访问 API 的 center IP/CIDR、XDP 初始模式。
- 安装完成后执行 guardd 进入节点维护菜单。
- 节点接入 center：在节点菜单导出 Base64 接入信息，复制到 center 的“添加节点”。

四、guardd-test 使用说明
- 安装命令同统一脚本，选择 3。
- 执行 guardd-test 进入测试菜单。
- 测试程序默认使用 veth 隔离实流测试或内核自测，不走公网，避免运营商提前拦截导致无效果。

五、Web 管理功能
- 总览大屏：实时流量趋势、节点状态、端口覆盖、最近 XDP 事件。
- 节点管理：添加/编辑节点、选择模板下发、观察/实时模式切换、XDP 重载/卸载、节点更新。
- 防御规则：规则库，只决定哪些规则可被模板选择；规则库修改不会自动影响已经复制到模板里的规则副本。
- 防护模板：规则 -> 模板 -> 节点。模板包含防护端口、白名单端口和规则副本。
- 端口画像：基于真实 flow_events 聚合端口、协议、TCP Flags、访问趋势。
- 流量明细：展示 center 从节点拉取入库的 XDP/L3/L4 包事件，包含源 IP、目标 IP/端口、协议、动作、原因、GeoIP、ASN。
- TOP 攻击源：按源 IP 聚合请求、拦截、观察命中、协议、端口、原因。
- 漏量评估：根据 guardd 节点实际看到的攻击流量估算硬件防火墙漏量。
- 系统管理：center 版本、更新、Web 配置、管理员密码、存储后端、GeoIP 资源状态。

六、快捷命令
安装后直接执行：
- guardd        进入节点维护菜单
- guardd-center 进入中心维护菜单
- guardd-test   进入节点测试工具

七、覆盖下载地址
可在执行安装脚本前覆盖：
export GUARDD_PACKAGE_URL=https://your-url/guardd-linux-amd64.tar.gz
export GUARDD_CENTER_PACKAGE_URL=https://your-url/guardd-center-linux-amd64.tar.gz
export GUARDD_TEST_PACKAGE_URL=https://your-url/guardd-test-linux-amd64.tar.gz
export GEOIP_ASSETS_URL=https://your-url/geoip-assets.tar.gz
curl -fsSL https://github.com/1227cwx/tsycdn-guardd-release/releases/latest/download/install.sh | bash
