guardd-suite v0.1.0 内测版统一安装与维护说明
==================================================

一、统一安装命令
--------------------------------------------------
root 用户直接执行：

curl -fsSL https://github.com/1227cwx/tsycdn-guardd-release/releases/latest/download/install.sh | bash

普通用户执行：

curl -fsSL https://github.com/1227cwx/tsycdn-guardd-release/releases/latest/download/install.sh | sudo -E bash

执行后选择：

1) guardd 节点 Agent
2) guardd-center 中心管理平台
3) guardd-test 节点测试工具
0) 退出

二、安装包 URL
--------------------------------------------------
guardd:
https://github.com/1227cwx/tsycdn-guardd-release/releases/latest/download/guardd-linux-amd64.tar.gz

guardd-center:
https://github.com/1227cwx/tsycdn-guardd-release/releases/latest/download/guardd-center-linux-amd64.tar.gz

guardd-test:
https://github.com/1227cwx/tsycdn-guardd-release/releases/latest/download/guardd-test-linux-amd64.tar.gz

统一安装脚本：
https://github.com/1227cwx/tsycdn-guardd-release/releases/latest/download/install.sh

可通过环境变量覆盖：GUARDD_PACKAGE_URL、GUARDD_CENTER_PACKAGE_URL、GUARDD_TEST_PACKAGE_URL。

三、guardd 节点安装引导
--------------------------------------------------
会引导填写：防护网卡、节点名称、节点分组、API 监听 IP/端口、允许访问 API 的 center IP/CIDR、API Key、初始观察/实时模式、节点 API 公网/内网访问地址。

首次上线建议选择观察模式。安装完成后复制输出的节点 Base64 信息或 API 信息到 guardd-center 添加节点。

四、guardd-center 安装引导
--------------------------------------------------
会引导填写：Web 监听 IP、Web 监听端口、HTTP/HTTPS、管理员账号密码、MySQL、Redis、数据保留时间。

MySQL 必须测试通过后才进入 Redis 配置，测试内容包括连接、认证、建表、写入、读取。
Redis 必须测试通过后才写配置，测试内容包括 PING、写入、读取、删除。

生产配置使用 MySQL + Redis；SQLite 仅保留给开发调试。

五、Web 后台核心逻辑
--------------------------------------------------
v0.1.0 固定为：

规则库 -> 防护模板 -> 节点下发 -> 节点观察/实时防护 -> center 展示真实数据

节点只有两种执行模式：
1. 观察模式：命中规则只记录 would_drop，不真正丢包。
2. 实时防护：命中规则后真实 XDP_DROP。

防护强度、端口、白名单、规则参数全部由“防护模板”体现。节点只选择模板并下发。

六、快捷维护命令
--------------------------------------------------
节点维护：

guardd

中心维护：

guardd-center

测试工具：

guardd-test

这些命令进入交互式菜单，不使用 guardd status、guardd config set 这类子命令。
菜单顶部会显示当前版本、更新日期、构建提交。

七、更新说明
--------------------------------------------------
在 guardd 或 guardd-center 菜单中选择检查更新。如果发现新版本，会提示是否更新；确认后自动下载最新包、替换二进制并重启服务。
下载的临时安装包在无需更新、取消更新或更新完成后都会自动清理。

Web 后台节点列表也支持节点远程检查/执行更新；系统管理页支持 center 检查/执行更新。

八、卸载说明
--------------------------------------------------
卸载 guardd 或 guardd-center 时会询问是否删除配置和数据目录，不再默认保留，也不再默认删除。

九、guardd-test 测试工具
--------------------------------------------------
guardd-test 必须安装在 guardd 节点服务器本机，并使用 root 运行。
它不会向公网发起攻击流量，默认使用 XDP 内核自测，也支持 veth 隔离实流测试。

测试菜单包括：导入节点 Base64 信息、查看节点状态、切换测试引擎、TCP SYN、UDP、ICMP、Bad TCP Flags、混合测试、指标对比、卸载。

十、紧急恢复
--------------------------------------------------
如果节点网络异常，优先执行：

guardd

选择“卸载当前 XDP 程序，不删除 guardd”。

如果菜单不可用但还能登录服务器，可执行：

ip link set dev eth0 xdp off

如果网卡不是 eth0，先用 ip -br addr 查看网卡名。

注意：停止 guardd 服务不等同于卸载 XDP。最稳妥的紧急恢复是卸载 XDP。
